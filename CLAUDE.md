# Paper Round

Daily literature monitor. A cloud routine runs screen.py's fetch/dedup,
screens new papers against the user's written scope, emails a digest via
Resend, and files keepers into Zotero. You are usually here to set it up
for the user or to adjust their scope.

## Files

| File | Role |
| --- | --- |
| `screen.py` | The pipeline. Contains no user settings; do not edit during setup |
| `config.yaml` | All user settings. Edit this, not screen.py |
| `.env` | Keys (from `.env.example`). Gitignored; never commit keys anywhere |
| `source_list.md` | The user's journals. **Parsed by regex** — see format rule below |
| `research_scope.md`, `relevance_rules.md` | The screening prompt, in two halves |
| `routine_prompt.md` | The cloud routine's task. The integrity rules block at its top is load-bearing; never weaken or remove it |
| `paper_log.csv`, `seen_papers.txt` | Generated by runs. Don't hand-edit |

## Hard rules

- `load_feeds()` only recognises feeds written exactly as:
  `### N. Journal Name` followed by ``- **Feed:** `url` ``. Keep that
  format when editing source_list.md; everything else in the file is
  free-form documentation.
- Journals whose RSS is blocked go in `config.yaml` under
  `crossref_fallbacks` (AMS) or `openalex_sources` (Elsevier, Nature),
  not in source_list.md feed entries.
- Papers in digests and logs must come verbatim from fetched feed data.
  Never let a prompt edit soften this.

## Setting up for a user

Follow the Start-here prompt in README.md: interview them for journals
and scope (ask for an abstract AND a Zotero CSV export — an abstract
alone is too thin), fill the files, then walk the cloud steps one
confirmation at a time. Prefer asking over assuming.

## Testing

- `pip install -r requirements.txt && python screen.py --dry-run` — free,
  no keys, verifies feeds and dedup. Run it after any feed change.
- A full local run needs `ANTHROPIC_API_KEY`; the cloud routine doesn't.

## Style

Minimal edits. Match the plain, concrete voice of the README; no filler,
no em dashes. Keep the scaffold structure of the scope files when
drafting into them.
